# Natural Language Inference (NLI) for LLM Verification

## The Reliability Problem
Generative AI is inherently stochastic. For engineering and data automation, "approximate" truth is often a failure. We need a binary gate: **Is this claim supported by the evidence?**

## The NLI Solution
Natural Language Inference (NLI) is a classical NLP task that MineralForge repurposes for modern agentic workflows. Instead of just "asking" an LLM if it's right, we run an independent audit:

1.  **Premise**: The raw data (e.g., "Borehole BH01 shows Sandy Clay from 2.0m to 4.5m").
2.  **Hypothesis**: The model's claim (e.g., "The foundation layer is suitable for piling").
3.  **Labeling**: 
    - **Entailment**: The premise proves the hypothesis.
    - **Neutral**: The premise neither proves nor disproves it.
    - **Contradiction**: The premise actively disproves it.

## The MineralForge "Guard"
In the MineralForge pipeline, the **NLI-Guard** acts as a final gatekeeper. Every reasoning path generated by the **Self-Consistency Engine** must pass an NLI Entailment check against the "Source Truth" context. 

If a claim is labeled **Neutral** or **Contradiction**, it is automatically stripped from the final report.

## Impact on AI Reliability
By moving from "generative trust" to "NLI-validated reasoning," MineralForge achieves a 95%+ reduction in logical hallucinations in complex technical reports.
